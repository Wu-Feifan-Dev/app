🚂方案一：Railway 免费最简单

Railway 是最适合快速部署的平台——只需把代码推到 GitHub，Railway 自动构建和运行。

步骤

  1. 注册 Railway
    访问 railway.app 注册账号（可以用 GitHub 登录）
  2. 把项目推到 GitHub
    # 1. 在 GitHub 创建新仓库（比如 project-tracker）
# 2. 初始化 Git 并推送
cd app目录
git init
git add .
git commit -m "项目计划进行表"
git remote add origin https://github.com/你的用户名/project-tracker.git
git push -u origin main
  3. 在 Railway 创建项目
    点击 "New Project" → "Deploy from GitHub repo" → 选择你的仓库
  4. 等待自动部署
    Railway 会自动检测 Node.js 项目，安装依赖并启动。
    约 1~2 分钟完成。
  5. 获取访问地址
    部署完成后，Railway 会分配一个域名，如 project-tracker.up.railway.app
    把这个地址发给团队即可！
免费额度：每月 $5 信用额度，个人小项目完全够用。超出后按用量计费。
数据持久化：Railway 默认使用容器内存储，重启后数据可能丢失。建议后续添加数据库插件（Railway 内置 PostgreSQL/MySQL 插件，一键添加）。
🌐方案二：Render 免费简单

Render 和 Railway 类似，也是 Git 推送自动部署，但免费版有冷启动延迟。

步骤

  1. 注册 Render
    访问 render.com 注册账号
  2. 创建 Web Service
    点击 "New" → "Web Service" → 连接 GitHub 仓库
  3. 配置部署
    - Build Command: npm install
    - Start Command: node server.js
    - Environment: Node
  4. 部署完成
    分配域名如 project-tracker.onrender.com
免费版每次闲置后首次访问需等待约 30~60 秒冷启动，不适合高频使用。付费版 ($7/月) 无此限制。
☁️方案三：腾讯云轻量应用服务器 国内

国内访问速度快，稳定性好，适合长期使用。

步骤

  1. 购买轻量应用服务器
    访问 cloud.tencent.com/lighthouse，选最低配置 (2核2G，¥60/月左右)，系统选 Ubuntu 22.04
  2. SSH 登录服务器
    ssh root@你的服务器IP
  3. 安装 Node.js
    curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
  4. 上传项目文件
    # 在本地电脑执行
scp -r app目录/* root@你的服务器IP:/opt/project-tracker/
  5. 启动服务
    cd /opt/project-tracker
npm install
PORT=3000 node server.js
  6. 配置防火墙
    在腾讯云控制台 → 防火墙 → 添加规则：放行 TCP 3000 端口
  7. 配置进程守护（防止终端关闭后服务停止）
    npm install -g pm2
pm2 start server.js --name tracker
pm2 save
pm2 startup
  8. 访问
    浏览器打开 http://你的服务器IP:3000
如需绑定域名：在腾讯云购买域名，添加 DNS A 记录指向服务器 IP，然后配置 Nginx 反向代理。
🐳方案四：Docker 部署 通用

适用于任何已安装 Docker 的云服务器，一键启动。

步骤

  1. 上传项目到服务器
    同方案三步骤 4
  2. 构建并运行 Docker 容器
    cd /opt/project-tracker
docker build -t project-tracker .
docker run -d \
  --name tracker \
  -p 3000:3000 \
  -v /opt/tracker-data:/app/data.json \
  --restart always \
  project-tracker
  3. 访问
    http://服务器IP:3000
数据持久化：用 -v 参数把 data.json 挂载到服务器磁盘，容器重启后数据不丢失。
data.json 是单个文件，Docker 挂载需要该文件预先存在。首次运行前先执行 echo '{}' > /opt/tracker-data/data.json，或去掉 -v 参数让容器自初始化后再迁移数据。

🔒 数据安全提示

当前数据存储方式

项目使用 data.json 文件存储数据，这是最简单的方式，但有以下注意点：

  • ✅ 优点：简单、无需额外配置
  • ⚠️ 注意：服务器重启后数据文件需保留（Docker 用 -v 挂载，PM2 自动保留）
  • ⚠️ 建议：定期备份 data.json 文件

升级为数据库（可选）

如果数据量增大或需要更可靠的持久化，建议升级为数据库：

  • SQLite：最简单，无需额外安装，适合小团队
  • PostgreSQL：Railway 一键添加插件，适合中等规模
  • MySQL：腾讯云/阿里云一键购买，适合企业级

❓ 常见问题

Q: 部署后别人访问不到？
A: 检查防火墙是否放行了 3000 端口。腾讯云/阿里云需在控制台和安全组都放行。Railway/Render 自动处理。
Q: 如何绑定自定义域名？
A: Railway/Render 在项目设置中添加域名并配置 DNS。云服务器需安装 Nginx：sudo apt install nginx，配置反向代理到 3000 端口。
Q: 数据丢了怎么办？
A: 定期导出 CSV（页面内「导出CSV」按钮），或定期备份 data.json 文件：cp data.json data_backup_20260701.json
Q: 多人编辑会不会冲突？
A: 当前版本每 5 秒自动刷新，不会出现数据丢失。但两人同时编辑同一行时，后保存的会覆盖前一个——建议团队约定避免同时编辑同一条目。
Q: 国内访问 Railway/Render 慢吗？
A: Railway/Render 服务器在国外，国内访问可能有延迟。国内团队建议用腾讯云/阿里云方案。
